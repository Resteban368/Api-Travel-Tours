## Description

API NestJS para tours con base de datos vectorizada (PostgreSQL + pgvector). La app Flutter envía el JSON del tour; la API genera el embedding con OpenAI **text-embedding-3-large** y lo guarda. n8n puede usar `POST /tours/search` con un vector de búsqueda para obtener las mejores coincidencias.

### Variables de entorno

Copia `.env.example` a `.env` y rellena:

- `DATABASE_URL`: PostgreSQL con extensión **pgvector** (ej. `postgresql://user:pass@localhost:5432/agente_viajes`).
- `OPENAI_API_KEY`: API key de OpenAI para embeddings.

En la base de datos debe existir la extensión. La API la crea al arrancar con `CREATE EXTENSION IF NOT EXISTS vector`.

### Endpoints

Usa las tablas `tours_maestro` y `n8n_vectors` de tu schema.

| Método | Ruta | Descripción |
|--------|------|-------------|
| POST | `/tours` | Crear tour en `tours_maestro` y su vector en `n8n_vectors`. Body: `id_tour`, `nombre_tour`, `link_pdf?`, `estado?`, `sede_id?`. |
| GET | `/tours` | Listar todos los tours (para llenar datos en la app). |
| GET | `/tours/:id` | Obtener un tour por `id` (PK de `tours_maestro`). |
| PATCH | `/tours/:id` | Actualizar tour y regenerar embedding en `n8n_vectors`. |
| POST | `/tours/search` | Búsqueda por similitud en `n8n_vectors`. Body: `{ "embedding": [3072 números] }`, query: `?limit=10`. |
